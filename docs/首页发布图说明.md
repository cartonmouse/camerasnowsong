# 首页发布图说明

当前 GitHub Pages 第一版部署先只上传首页展示图，避免把全部原图或大尺寸 JPG 一次性放进 GitHub 仓库。

## 当前做法

- 首页 Star 且未隐藏的 9 张照片已导出为 WebP。
- 发布图路径为：

```text
public/site-photos/home/
```

- `src/data/photos.json` 中这 9 张照片的 `src` 已指向 `/site-photos/home/...webp`。
- 原始照片仍保留在本地 `public/photos/`，不提交到 GitHub。

## 注意

如果之后运行 `npm.cmd run photos:sync`，`src/data/photos.json` 可能会被重新生成，首页发布图路径也可能被覆盖。后续更稳定的方案是把发布图生成和路径写入纳入同步脚本，或者迁移到对象存储 / CDN。

## 后续方向

1. 先用少量压缩图让线上首页正常显示。
2. 再决定是否为作品页批量生成发布图。
3. 照片数量变多后，改为对象存储或 CDN 托管图片。
