# 个人摄影网站：首页照片管理阶段总结

日期：2026-06-22

## 背景

在相册管理工具完成后，网站已经可以通过本地页面管理相册名称、题材、描述，并把照片同步到主站。但首页展示仍然有一个明显问题：相册封面、首页精选和真实想放到首页的照片混在一起，语义不清晰。

因此，本阶段把“首页照片”从相册封面里拆出来，建立独立的首页管理流程。

## 本阶段目标

- 相册封面只表示相册代表图，不再决定首页展示。
- 在相册管理中可以给具体照片 Star。
- 首页只优先展示 Star 照片。
- 首页照片可以单独排序、隐藏、设置展示数量。
- 管理操作要有明确保存反馈，避免不知道是否点中了按钮。

## 新增核心概念

### Star

Star 表示“这张照片可以进入首页照片池”。

它保存在对应相册的 `album.json` 中：

```json
{
  "photos": {
    "IMG_0001.jpg": {
      "star": true
    }
  }
}
```

### homepage.json

首页排序不写进各个相册，而是集中保存在：

```text
public/photos/homepage.json
```

它负责：

- 首页照片顺序
- 某张 Star 照片是否隐藏
- 首页展示数量

示例：

```json
{
  "items": [
    {
      "src": "/photos/example/IMG_0001.jpg",
      "hidden": false
    }
  ],
  "limit": 12
}
```

## 管理页面变化

管理页面现在有两个主要模块：

- 相册管理
- 首页照片管理

### 相册管理

负责：

- 相册名称
- 相册描述
- 题材分类
- 给照片 Star / 取消 Star

保存按钮：

```text
保存相册信息
```

点击后会显示：

- 正在保存相册信息...
- 保存中...
- 保存成功或失败提示

### 首页照片管理

负责：

- 置顶
- 置底
- 上移
- 下移
- 隐藏 / 显示
- 取消首页展示
- 首页展示数量

保存按钮：

```text
保存首页顺序
```

保存后仍需点击：

```text
更新主站
```

主网页才会读取最新生成的 `src/data/photos.json`。

## 数据流

当前完整数据流如下：

```text
public/photos/真实照片
→ album.json 记录相册和 Star 信息
→ homepage.json 记录首页排序和隐藏信息
→ 点击“更新主站”或运行 photos:sync
→ 生成 src/data/photos.json
→ React 主站读取 photos.json
→ 首页按 Star + homepage.json 展示
```

## 已解决的问题

- 首页展示不再依赖相册封面。
- 可以明确挑选首页照片。
- 可以控制首页照片顺序。
- 不需要一格一格慢慢移动照片，新增了置顶和置底。
- 可以临时隐藏首页照片，而不取消 Star。
- 保存操作有即时反馈。
- 首页管理数据和相册管理数据职责分离。

## 当前注意事项

- 在相册管理中修改 Star 后，要点击“保存相册信息”。
- 在首页照片管理中改顺序、隐藏状态或展示数量后，要点击“保存首页顺序”。
- 想让主网页变化，最后都要点击“更新主站”。
- 如果主站页面没变化，通常是忘了更新主站或刷新主网页。

## 后续建议

下一阶段最值得做的是“单张照片信息管理”。目前单张照片标题、描述、地点、年份仍主要依赖手动编辑 `album.json`。如果管理页能直接编辑单张照片信息，日常维护会更完整。
