# 个人摄影网站使用手册

更新日期：2026-06-23

这份手册记录个人摄影网站的日常使用流程。目标是：新增照片、整理相册、编辑单张照片信息、挑选首页照片时，尽量通过本地管理页面完成，不再频繁手动编辑 JSON 文件。

## 项目位置

真实项目目录：

```text
D:\3BUPT\mark'workshop\personal-photography-website-superpowers
```

照片目录：

```text
D:\3BUPT\mark'workshop\personal-photography-website-superpowers\public\photos
```

每个相册对应 `public/photos/` 下的一个文件夹。

## 打开网站与管理工具

进入项目目录：

```powershell
cd "D:\3BUPT\mark'workshop\personal-photography-website-superpowers"
```

启动主网站：

```powershell
npm.cmd run dev
```

打开主网站：

```text
http://127.0.0.1:5173/
```

启动本地管理工具：

```powershell
npm.cmd run admin
```

打开管理页面：

```text
http://127.0.0.1:4000/admin
```

如果浏览器显示 `ERR_CONNECTION_REFUSED`，通常表示对应服务没有启动。主网站打不开就启动 `npm.cmd run dev`；管理页面打不开就启动 `npm.cmd run admin`。

## 日常新增照片流程

1. 把新的照片文件夹放入：

```text
public/photos/
```

2. 启动管理工具：

```powershell
npm.cmd run admin
```

3. 打开：

```text
http://127.0.0.1:4000/admin
```

4. 点击“扫描”。

扫描会读取 `public/photos/` 下的照片文件夹，并为缺少配置的相册生成 `album.json`。

5. 在“相册管理”中填写相册信息：

- 相册名称
- 相册描述
- 题材分类
- 相册封面

6. 如果某张照片需要单独说明，可以在照片卡片里填写：

- 标题
- 描述
- 地点
- 年份

7. 如果某张照片想放到首页，给它点 Star。

8. 点击“保存相册信息”。

保存成功后，页面会显示保存成功提示。

9. 进入“首页照片管理”，调整首页照片：

- 置顶
- 置底
- 上移
- 下移
- 隐藏 / 显示
- 取消首页展示
- 展示数量

10. 点击“保存首页顺序”。

11. 点击“更新主站”。

这一步会重新生成：

```text
src/data/photos.json
```

主网站真正读取的是这个文件。

12. 刷新主网站：

```text
http://127.0.0.1:5173/
```

## 推荐工作流

最常用顺序：

```text
放入照片文件夹
-> 打开 admin 管理页
-> 扫描
-> 填写相册信息
-> 选择相册封面
-> 编辑需要单独说明的照片
-> 给首页照片点 Star
-> 保存相册信息
-> 进入首页照片管理
-> 调整首页顺序和展示数量
-> 保存首页顺序
-> 更新主站
-> 刷新主网站
```

## Star、封面、精选的区别

### Star

Star 表示“这张照片可以进入首页照片池”。

现在首页照片管理主要依赖 Star。

### 封面

封面表示相册代表图。它只影响相册卡片和相册识别，不再自动决定首页展示。

### 精选

精选是早期保留下来的兼容机制。当没有任何 Star 照片时，主站会使用旧的精选规则，避免首页空白。正常使用时，主要管理 Star 和首页顺序即可。

## 相册管理

相册管理负责：

- 相册名称
- 相册描述
- 题材分类
- 相册封面
- 单张照片标题
- 单张照片描述
- 单张照片地点
- 单张照片年份
- Star / 取消 Star

修改这些内容后，需要点击：

```text
保存相册信息
```

如果保存成功，页面会出现保存成功提示。

## 首页照片管理

首页照片管理只管理已经 Star 的照片。

常用按钮：

- “置顶”：把照片直接放到首页最前面。
- “置底”：把照片直接放到最后。
- “上移”：向前移动一格。
- “下移”：向后移动一格。
- “隐藏”：保留 Star，但暂时不在首页显示。
- “显示”：恢复隐藏照片。
- “取消首页展示”：取消这张照片的 Star，并写回对应相册的 `album.json`。

注意：

- 调整顺序、隐藏状态、展示数量后，需要点击“保存首页顺序”。
- 保存首页顺序后，还需要点击“更新主站”，主网站才会变化。

## 文件关系

### public/photos/

存放真实照片。每个子文件夹是一组相册。

### album.json

放在每个相册文件夹里，记录相册标题、题材、描述、封面，以及单张照片独立信息。照片的 Star 状态也保存在这里。

### homepage.json

放在 `public/photos/` 下，记录首页照片顺序、隐藏状态和展示数量。它只负责首页，不负责相册内容。

### src/data/photos.json

主站真正读取的数据文件。点击管理页里的“更新主站”或运行 `npm.cmd run photos:sync` 后会重新生成。

### admin/

本地相册管理页面的前端文件。

### scripts/admin-server.mjs

本地相册管理工具的服务端入口。运行 `npm.cmd run admin` 时会启动它。

## album.json 基本格式

```json
{
  "title": "相册名称",
  "topics": ["风景", "旅行"],
  "description": "这组照片的说明。",
  "featured": true,
  "cover": "IMG_0001.jpg",
  "photos": {}
}
```

通常只需要在管理页面填写，不需要手动改这个文件。

如果某张照片需要单独标题或描述，管理页面会在 `photos` 里写入类似内容：

```json
{
  "photos": {
    "IMG_0001.jpg": {
      "title": "单张照片标题",
      "description": "单张照片说明",
      "location": "北京",
      "year": "2026",
      "star": true
    }
  }
}
```

如果某张照片没有任何单独信息，也没有 Star，它不会长期保留空配置。

## homepage.json 基本格式

```json
{
  "items": [
    {
      "src": "/photos/example/IMG_0001.jpg",
      "hidden": false
    }
  ],
  "limit": 12
}
```

通常不要手动编辑它，直接用“首页照片管理”即可。

## 关闭本地管理服务

如果知道 PID，可以运行：

```powershell
Stop-Process -Id 进程编号
```

例如：

```powershell
Stop-Process -Id 23540
```

PID 每次启动都可能不同。

也可以关闭对应 PowerShell 窗口，或者之后再让 Codex 帮你查找并关闭占用 `4000` 端口的进程。

## 常见问题

### 打不开 http://127.0.0.1:4000/admin

通常是管理服务没有运行。进入项目目录后运行：

```powershell
npm.cmd run admin
```

### 打不开 http://127.0.0.1:5173/

通常是主站开发服务器没有运行。进入项目目录后运行：

```powershell
npm.cmd run dev
```

### 管理页保存后主站没有变化

需要点击管理页里的“更新主站”，或者在终端运行：

```powershell
npm.cmd run photos:sync
```

然后刷新主站页面。

### Star 后首页没有出现这张照片

检查四件事：

1. 是否点击过“保存相册信息”。
2. 是否在“首页照片管理”中被隐藏。
3. 是否点击过“保存首页顺序”。
4. 是否点击过“更新主站”并刷新主网页。

### 首页顺序没有变化

调整顺序后需要点击“保存首页顺序”，再点击“更新主站”。

### 新照片没有出现在题材分类里

检查对应相册是否已经选择题材，并确认点击过“保存相册信息”和“更新主站”。

### 管理页面预览图破图

如果缩略图显示为破图，通常是管理服务没有正确托管 `/photos/...` 图片资源，或者图片路径中对应文件不存在。先刷新管理页面；如果仍然破图，再重启 `npm.cmd run admin`。

### Git 提示 LF will be replaced by CRLF

这是 Windows 下的换行符提示，不是代码错误。它表示 Git 以后可能把文件换行从 LF 转成 CRLF。当前可以先继续使用；如果以后需要团队协作规范，可以增加 `.gitattributes`。
