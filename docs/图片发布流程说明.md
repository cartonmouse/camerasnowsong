# 图片发布流程说明

更新日期：2026-06-24

## 当前目标

网站源码可以部署到 GitHub Pages，但大量原图不适合直接作为网页资源发布。当前新增的发布流程用于把 `public/photos/` 中的本地照片生成适合网页访问的 WebP 图片。

## 生成内容

运行发布脚本后，每张可发布照片会生成两张 WebP：

- `thumb`：列表和相册网格使用，最长边 720px。
- `src`：点开 Lightbox 后使用，最长边 2200px。

输出目录：

```text
public/site-photos/albums/
```

主站数据仍然读取：

```text
src/data/photos.json
```

发布脚本会在 `photos.json` 中补充：

- `originalSrc`：原始本地照片路径。
- `thumb`：发布用缩略图路径。
- `src`：发布用大图路径。
- `thumbWidth` / `thumbHeight`：缩略图尺寸。
- `width` / `height`：大图尺寸。

## 常用命令

完整刷新照片数据并生成发布图：

```powershell
npm.cmd run photos:sync
npm.cmd run photos:publish
```

强制重新生成已经存在的 WebP：

```powershell
npm.cmd run photos:publish:force
```

验证：

```powershell
npm.cmd run test:data
npm.cmd run build
```

## 前端读取规则

- 照片网格优先读取 `photo.thumb`。
- 如果没有 `thumb`，回退到 `photo.src`。
- Lightbox 继续读取 `photo.src`。

这意味着列表加载更轻，点开后可以显示更清晰的大图。

## 注意事项

- `photos:sync` 会重新生成 `src/data/photos.json`，因此运行后需要再运行 `photos:publish`。
- 目前发布脚本只处理 `photos.json` 中指向 `/photos/...` 的本地图片。
- 之前首页临时发布图位于 `public/site-photos/home/`，当前正式流程已替换为 `public/site-photos/albums/`。
- 不建议提交 RAW、CR2 或未经压缩的大图到 GitHub。
