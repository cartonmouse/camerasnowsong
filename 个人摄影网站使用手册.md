# 个人摄影网站使用手册

更新时间：2026-06-22

这份手册记录个人摄影网站的日常使用流程。目标是：新增照片、整理相册、挑选首页照片时，尽量通过本地管理页面完成，不再频繁手动编辑 JSON 文件。

## 项目位置

真实项目目录：

```text
D:\3BUPT\mark'workshop\personal-photography-website-superpowers
```

照片目录：

```text
D:\3BUPT\mark'workshop\personal-photography-website-superpowers\public\photos
```

每个相册对应 `public/photos/` 下的一个文件夹。

## 日常新增照片流程

1. 把新的照片文件夹放入：

```text
public/photos/
```

2. 打开 PowerShell，进入项目目录：

```powershell
cd "D:\3BUPT\mark'workshop\personal-photography-website-superpowers"
```

3. 启动本地相册管理工具：

```powershell
npm.cmd run admin
```

4. 打开管理页面：

```text
http://127.0.0.1:4000/admin
```

5. 点击“扫描”。

这一步会扫描 `public/photos/`，并给缺少配置的新相册文件夹生成 `album.json`。

6. 在“相册管理”中填写相册信息：

- 相册名称
- 相册描述
- 题材分类
- 首页回退精选状态
- 需要在首页展示的照片 Star

7. 点击“保存相册信息”。

这一步会把相册信息和照片 Star 状态写入对应相册文件夹里的 `album.json`。

8. 如果要控制首页照片顺序，进入“首页照片管理”：

- 用“置顶 / 置底”快速调整大范围顺序
- 用“上移 / 下移”做细调
- 用“隐藏 / 显示”临时控制某张 Star 照片是否出现在首页
- 修改“展示数量”
- 点击“保存首页顺序”

这一步会写入：

```text
public/photos/homepage.json
```

9. 点击“更新主站”。

这一步会重新生成：

```text
src/data/photos.json
```

主站真正读取的是这个文件。

10. 打开或刷新主站：

```text
http://127.0.0.1:5173/
```

如果主站没有运行，可以启动：

```powershell
npm.cmd run dev
```

## 推荐工作流

最常用顺序：

```text
放入照片文件夹
→ 打开 admin 管理页
→ 点击“扫描”
→ 填写相册信息
→ 给首页照片点 Star
→ 点击“保存相册信息”
→ 进入“首页照片管理”
→ 调整首页顺序和展示数量
→ 点击“保存首页顺序”
→ 点击“更新主站”
→ 刷新主站
```

## Star、封面、精选的区别

`Star`：

用于决定某张照片是否进入首页照片池。现在首页照片管理主要依赖 Star。

`封面`：

表示相册代表图。它不再自动决定首页展示。

`首页回退精选`：

当没有任何 Star 照片时，主站会使用旧的精选规则，避免首页空白。正常使用时，你主要管理 Star 和首页顺序即可。

## 首页照片管理

首页照片管理只管理已经 Star 的照片。

常用按钮：

- “置顶”：把照片直接放到首页最前面。
- “置底”：把照片直接放到最后。
- “上移”：向前移动一格，适合细调。
- “下移”：向后移动一格，适合细调。
- “隐藏”：保留 Star，但暂时不在首页展示。
- “显示”：恢复隐藏照片。
- “取消首页展示”：取消这张照片的 Star，并写回对应相册的 `album.json`。

注意：

- 调整顺序、隐藏状态、展示数量后，需要点击“保存首页顺序”。
- 保存首页顺序后，还需要点击“更新主站”，主网页才会变化。

## 什么时候点击哪些保存按钮

在“相册管理”里改了这些内容后，点击“保存相册信息”：

- 相册名称
- 相册描述
- 题材分类
- Star / 取消 Star

在“首页照片管理”里改了这些内容后，点击“保存首页顺序”：

- 置顶
- 置底
- 上移
- 下移
- 隐藏 / 显示
- 展示数量

最后只要希望主网页反映最新内容，就点击“更新主站”。

## 文件关系

`public/photos/`：

存放真实照片。每个子文件夹是一组相册。

`album.json`：

放在每个相册文件夹里，记录这个相册的标题、题材、描述、精选状态，以及可选的单张照片独立信息。照片的 Star 状态也保存在这里。

`homepage.json`：

放在 `public/photos/` 下，记录首页照片顺序、隐藏状态和展示数量。它只负责首页，不负责相册内容。

`src/data/photos.json`：

主站真正读取的数据文件。点击管理页里的“更新主站”或运行 `npm.cmd run photos:sync` 后会重新生成。

`admin/`：

本地相册管理页面的前端文件。

`scripts/admin-server.mjs`：

本地相册管理工具的服务端入口。运行 `npm.cmd run admin` 时会启动它。

## album.json 基本格式

```json
{
  "title": "相册名称",
  "topics": ["风景", "旅行"],
  "description": "这组照片的说明。",
  "featured": true,
  "photos": {}
}
```

通常只需要在管理页面填写相册级信息，不需要手动修改这个文件。

如果某张照片需要单独标题或描述，可以在 `photos` 里补充：

```json
{
  "photos": {
    "IMG_0001.jpg": {
      "title": "单张照片标题",
      "description": "单张照片说明",
      "topics": ["风景"],
      "location": "北京",
      "featured": true,
      "star": true
    }
  }
}
```

管理工具保存相册信息时会保留已有的单张照片配置。

## homepage.json 基本格式

```json
{
  "items": [
    {
      "src": "/photos/example/IMG_0001.jpg",
      "hidden": false
    }
  ],
  "limit": 12
}
```

通常不要手动编辑它，直接用“首页照片管理”即可。

## 关闭本地管理服务

如果知道 PID，可以运行：

```powershell
Stop-Process -Id 进程编号
```

例如：

```powershell
Stop-Process -Id 23540
```

PID 每次启动都可能不同。

也可以关闭对应的 PowerShell 窗口，或之后再让 Codex 帮你查找并关闭占用 `4000` 端口的进程。

## 常见问题

### 打不开 http://127.0.0.1:4000/admin

通常是管理服务没有运行。进入项目目录后运行：

```powershell
npm.cmd run admin
```

### 打不开 http://127.0.0.1:5173/

通常是主站开发服务器没有运行。进入项目目录后运行：

```powershell
npm.cmd run dev
```

### 管理页保存后主站没有变化

需要点击管理页里的“更新主站”，或者在终端运行：

```powershell
npm.cmd run photos:sync
```

然后刷新主站页面。

### Star 后首页没有出现这张照片

检查四件事：

1. 是否点击过“保存相册信息”。
2. 是否在“首页照片管理”中被隐藏。
3. 是否点击过“保存首页顺序”。
4. 是否点击过“更新主站”并刷新主网页。

### 首页顺序没变化

调整顺序后需要点击“保存首页顺序”，再点击“更新主站”。

### 新照片没有出现在题材分类里

检查对应相册是否已经选择题材，并确认点击过“保存相册信息”和“更新主站”。
